در این کارگاه، فرآیند **طراحی، خودکارسازی و اجرای تست‌های API** بدون وابستگی به واسط کاربری (UI) به‌صورت عملی آموزش داده می‌شود.
::

## معرفی کارگاه

تست‌های API یکی از مهم‌ترین شاخص‌های ارزیابی **عملکرد صحیح هسته سامانه** هستند.  
در این نوع تست‌ها، بدون فرض وجود واسط گرافیکی (وب یا موبایل)، ماژول‌های مختلف سامانه از طریق **واسط‌های برنامه‌نویسی (API)** مورد بررسی قرار می‌گیرند؛ که این واسط‌ها معمولاً به‌صورت **RESTful** یا **SOAP** پیاده‌سازی شده‌اند.

در این کارگاه، تست‌های API در دو دسته‌ی اصلی زیر طراحی و پیاده‌سازی می‌شوند:

- **CRUD**: ارزیابی عملیات ساخت، بازیابی، ویرایش و حذف موجودیت‌های سامانه  
- **BDD**: ارزیابی رفتار سامانه بر اساس سناریوهای رفتارمحور  


## مراحل کلی کارگاه

::steps{level="3"}

### آشنایی با مفاهیم و ساختار API
درک مفاهیم پایه API و Rest API و نحوه فراخوانی سرویس‌ها و تحلیل پاسخ‌ها.

### طراحی و پیاده‌سازی سناریوهای تست
طراحی سناریوهای CRUD و رفتارمحور (BDD) و پیاده‌سازی آن‌ها با ابزارهای متداول.

### اعتبارسنجی و خودکارسازی پیشرفته
ایجاد Assertionها، اجرای تست‌ها به‌صورت گروهی و Headless و پیاده‌سازی در ابزارهای حرفه‌ای.

::

## سرفصل‌های کارگاه

::accordion

:::accordion-item{label="مفاهیم پایه"}
- مفاهیم پایه API  
- مفاهیم Rest API  
- بررسی نحوه فراخوانی APIها و دریافت پاسخ  
:::

:::accordion-item{label="سناریوهای CRUD"}
- ساخت (Create) موجودیت‌ها  
- بازیابی (Read) اطلاعات  
- ویرایش (Update) داده‌ها  
- حذف (Delete) موجودیت‌های منطقی سامانه  
:::

:::accordion-item{label="اعتبارسنجی پاسخ‌ها"}
- انواع کدهای وضعیت (Status Codes)  
- استخراج داده‌ها از JSON دریافتی  
- ایجاد انواع Assertionها برای اعتبارسنجی پاسخ‌ها  
:::

:::accordion-item{label="سناریوهای رفتارمحور (BDD)"}
- مفاهیم طراحی سناریوهای رفتارمحور  
- آشنایی با ابزار Cucumber  
- راه‌حل ترکیبی Postman و Cucumber برای پیاده‌سازی سناریوهای BDD  
:::

:::accordion-item{label="مفاهیم پیشرفته در Postman"}
- طراحی تست‌های ساختارمند  
- ایجاد جریان کاری بین چند درخواست (Data Flow)  
- کدهای سراسری (Global Scripts) برای استفاده مجدد  
- اجرای دسته‌جمعی تست‌ها  
- اجرای تست‌ها در حالت Headless و محیط خط فرمان  
:::

:::accordion-item{label="پیاده‌سازی تست‌های API در JMeter"}
- بررسی مفاهیم اصلی تست API در JMeter  
- اتصال داده‌ها برای سناریوهای داده‌محور  
- اجرای تست‌ها در حالت Headless و محیط دستوری  
:::
